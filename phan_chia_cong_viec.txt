================================================================================
          PHÂN CHIA CÔNG VIỆC DỰ ÁN MESS APP - 6 THÀNH VIÊN
================================================================================

Dự án Mess App được xây dựng bằng Backend FastAPI (Python) và Frontend React/Vite (Javascript).
Dưới đây là bảng phân chia công việc chi tiết được phân chia theo độ khó từ khó nhất đến dễ nhất, giao cho các thành viên trong nhóm:
1. Trần Lê Hoàng (Phần khó nhất - Module 6: Upload, Tìm kiếm & Realtime Socket)
2. Đỗ Ngọc Thịnh (Module 5: Tin nhắn & Tương tác tin nhắn)
3. Phạm Đình Quang (Module 4: Quản lý cuộc hội thoại)
4. Nguyễn Đức Anh (Module 3: Bạn bè & Yêu cầu kết bạn)
5. Nguyễn Khắc Hiếu (Module 1: Đăng nhập & Đăng ký)
6. Phạm Quốc Huy (Phần dễ nhất - Module 2: Hồ sơ cá nhân)

--------------------------------------------------------------------------------
THÀNH VIÊN 1: NGUYỄN KHẮC HIẾU - MODULE ĐĂNG NHẬP & ĐĂNG KÝ (AUTH & CORS)
--------------------------------------------------------------------------------
* BẢNG ĐÁNH GIÁ ĐÓNG GÓP:
STT	Họ và tên thành viên	Nhiệm vụ / phần việc	Tỷ lệ đóng góp (%)
1	Nguyễn Khắc Hiếu		Đăng ký, Đăng nhập, JWT, bcrypt, FE Login/Register, AuthShell, DB Users	100 %
TỔNG CỘNG	100%

* Nhiệm vụ chính: Xây dựng hệ thống xác thực người dùng ban đầu của ứng dụng.
* Backend (FastAPI):
  - Viết API đăng ký tài khoản: POST `/auth/register`
  - Viết API đăng nhập tài khoản: POST `/auth/login`
  - Xử lý mã hóa mật khẩu bằng thư viện `bcrypt`
  - Thiết lập cơ chế tạo và xác thực Token JWT (`utils/jwt_helper.py`)
  - Thiết lập cấu hình CORS Middleware cho phép kết nối chéo cổng giữa FE và BE.
* Frontend (React):
  - Thiết kế và phát triển giao diện trang đăng nhập: `Login.jsx`
  - Thiết kế và phát triển giao diện trang đăng ký: `Register.jsx`
  - Tạo layout bao bọc dùng chung cho trang đăng nhập/đăng ký: `AuthShell.jsx`
  - Thiết lập lớp gọi API (`api/authApi.js`) để kết nối POST lên backend.
  - Lưu trữ Token JWT vào `localStorage` sau khi đăng nhập/đăng ký thành công.
* Database:
  - Bảng `users`: Định nghĩa các trường dữ liệu thiết yếu (`id`, `phone`, `password_hash`, `full_name`, `status` online/offline).

THÀNH VIÊN 2: PHẠM QUỐC HUY - MODULE QUẢN LÝ TÀI KHOẢN & HỒ SƠ CÁ NHÂN (USER PROFILE)
--------------------------------------------------------------------------------
* BẢNG ĐÁNH GIÁ ĐÓNG GÓP:
STT	Họ và tên thành viên	Nhiệm vụ / phần việc	Tỷ lệ đóng góp (%)
1	Phạm Quốc Huy		API Profile/update, đếm bạn bè, tương tác nhiều nhất, FE Profile Drawer/Edit profile	100 %
TỔNG CỘNG	100%

* Nhiệm vụ chính: Phát triển trang quản lý thông tin người dùng và thống kê tương tác.
* Backend (FastAPI):
  - Viết API lấy thông tin hồ sơ: GET `/auth/profile`
    + Yêu cầu thống kê: Đếm số lượng bạn bè hiện có.
    + Thống kê nâng cao: Lấy danh sách tối đa 3 bạn bè tương tác (nhắn tin) nhiều nhất.
  - Viết API cập nhật hồ sơ cá nhân: PUT `/auth/profile` (cập nhật họ tên, ảnh đại diện, ngày sinh).
* Frontend (React):
  - Thiết kế Drawer thông tin cá nhân (`profile-drawer` trượt từ cạnh trái màn hình ra).
  - Thiết kế Modal chỉnh sửa thông tin cá nhân (Họ tên, ngày sinh, đường dẫn avatar).
  - Kết nối gọi API lấy/cập nhật hồ sơ từ backend.
* Database:
  - Bảng `users`: Bổ sung các cột thông tin chi tiết (`dob` - ngày sinh, `avatar` - đường dẫn ảnh đại diện).

THÀNH VIÊN 3: NGUYỄN ĐỨC ANH - MODULE BẠN BÈ & YÊU CẦU KẾT BẠN (FRIENDSHIP)
--------------------------------------------------------------------------------
* BẢNG ĐÁNH GIÁ ĐÓNG GÓP:
STT	Họ và tên thành viên	Nhiệm vụ / phần việc	Tỷ lệ đóng góp (%)
1	Nguyễn Đức Anh		API friend request/accept/reject/list/block/remove, FE Search Friend Modal, RightPanel requests, sidebar friends	100 %
TỔNG CỘNG	100%

* Nhiệm vụ chính: Quản lý mối quan hệ bạn bè, gửi lời mời và danh sách chặn.
* Backend (FastAPI):
  - Viết API gửi yêu cầu kết bạn: POST `/friend/request` (hoặc POST `/friend/add`)
  - Viết API chấp nhận kết bạn: PUT `/friend/accept/{id}` (khi chấp nhận kết bạn thì tự động tạo cuộc hội thoại private)
  - Viết API từ chối kết bạn: PUT `/friend/reject/{id}` (hoặc DELETE)
  - Viết API danh sách bạn bè: GET `/friend/list/{user_id}`
  - Viết API chặn người dùng: PUT `/friend/block/{friend_id}`
  - Viết API xóa bạn bè: DELETE `/friend/{friend_id}`
* Frontend (React):
  - Thiết kế Modal tìm kiếm người dùng qua số điện thoại để kết bạn.
  - Phát triển bảng hiển thị "Lời mời kết bạn mới" ở khu vực cột bên phải (`RightPanel.jsx`).
  - Thiết lập danh sách bạn bè trên Sidebar (Tab "Bạn bè").
  - Xử lý các nút bấm hành động: Kết bạn, Chấp nhận, Từ chối, Hủy kết bạn, Chặn.
* Database:
  - Bảng `friendships`: Quản lý mối liên kết bạn bè với các trạng thái (`pending` - đang chờ, `accepted` - đã kết bạn, `blocked` - đã chặn) và các cột `user_id`, `friend_id`.

THÀNH VIÊN 4: PHẠM ĐÌNH QUANG - MODULE QUẢN LÝ CUỘC HỘI THOẠI (CONVERSATIONS)
--------------------------------------------------------------------------------
* BẢNG ĐÁNH GIÁ ĐÓNG GÓP:
STT	Họ và tên thành viên	Nhiệm vụ / phần việc	Tỷ lệ đóng góp (%)
1	Phạm Đình Quang		API private/group chats, list conversations, FE Sidebar, Tab all/groups, Online status, active chat selection	100 %
TỔNG CỘNG	100%

* Nhiệm vụ chính: Xử lý danh sách chat, phân chia chat đơn/chat nhóm.
* Backend (FastAPI):
  - Viết API tạo cuộc trò chuyện cá nhân: POST `/conversation/private` (kiểm tra nếu đã tồn tại cuộc chat giữa 2 người thì không tạo trùng)
  - Viết API tạo nhóm chat: POST `/conversation/group` (chứa tên nhóm, danh sách ID các thành viên)
  - Viết API lấy danh sách cuộc hội thoại: GET `/conversation/list`
    + Yêu cầu: Trích xuất nội dung tin nhắn cuối cùng (`last_message`), thời gian gửi (`last_time`), ID người gửi tin cuối (`last_message_sender_id`) và trạng thái tin cuối để đồng bộ hiển thị tích.
* Frontend (React):
  - Thiết kế Sidebar danh sách hội thoại hiển thị ở bên trái ứng dụng.
  - Phân loại tab hội thoại: Tab "Tất cả" (hiển thị mọi chat) và Tab "Nhóm" (chỉ lọc chat nhóm).
  - Tích hợp vòng tròn trạng thái online/offline của đối phương trên ảnh đại diện (`Avatar`).
  - Thiết lập logic bấm chọn cuộc hội thoại active để chuyển đổi khung chat ở giữa.
* Database:
  - Bảng `conversations`: Quản lý danh sách hội thoại (`id`, `type`: private/group, `name`, `avatar`, `created_by`).
  - Bảng `conversation_members`: Quản lý thành viên tham gia từng cuộc hội thoại (`conversation_id`, `user_id`).

THÀNH VIÊN 5: ĐỖ NGỌC THỊNH - MODULE TIN NHẮN & TƯƠNG TÁC TIN NHẮN (MESSAGING & UI WINDOW)
--------------------------------------------------------------------------------
* BẢNG ĐÁNH GIÁ ĐÓNG GÓP:
STT	Họ và tên thành viên	Nhiệm vụ / phần việc	Tỷ lệ đóng góp (%)
1	Đỗ Ngọc Thịnh		API message send/list/recall/delete/pin, FE ChatHeader, MessageList/Input, MessageBubble (Reply, Recall, Delete, Status ticks)	100 %
TỔNG CỘNG	100%

* Nhiệm vụ chính: Xử lý luồng tin nhắn chính trong cuộc trò chuyện và các tương tác phụ.
* Backend (FastAPI):
  - Viết API gửi tin nhắn mới: POST `/message/send` (hỗ trợ loại tin nhắn text, image, file và trả lời tin nhắn khác)
  - Viết API lấy lịch sử tin nhắn: GET `/message/list` (kèm tham số `conversation_id`, tự động chuyển trạng thái tin nhắn nhận được thành `seen` nếu người gọi API là người nhận)
  - Viết API thu hồi tin nhắn: PUT `/message/recall/{message_id}` (đổi trạng thái `is_recalled = True`)
  - Viết API xóa tin nhắn: DELETE `/message/{message_id}` (hoặc ẩn đi đối với người dùng)
  - Viết API ghim tin nhắn: PUT `/message/pin/{message_id}`
* Frontend (React):
  - Thiết kế khung hiển thị hội thoại chính (`Chat.jsx` kết nối với `ChatHeader`, `MessageList`, `MessageInput`).
  - Vẽ giao diện bong bóng tin nhắn (`MessageBubble.jsx` phân biệt tin nhắn của mình ở bên phải và đối phương bên trái).
  - Thiết kế cụm nút chức năng khi click vào tin nhắn: Trả lời (Reply), Thu hồi (Recall), Xóa (Delete).
  - Tích hợp logic trạng thái tin nhắn nhỏ ở góc tin nhắn cuối cùng (`· Đã gửi`, `· Đã nhận`, `· Đã xem`).
* Database:
  - Bảng `messages`: Quản lý chi tiết tin nhắn (`id`, `conversation_id`, `sender_id`, `content`, `message_type`, `file_url`, `is_recalled`, `reply_message_id`, `status` [sent, delivered, seen], `created_at`).

THÀNH VIÊN 6: TRẦN LÊ HOÀNG - MODULE TÍNH NĂNG NÂNG CAO (UPLOAD, SEARCH & REALTIME SOCKET)
--------------------------------------------------------------------------------
* BẢNG ĐÁNH GIÁ ĐÓNG GÓP:
STT	Họ và tên thành viên	Nhiệm vụ / phần việc	Tỷ lệ đóng góp (%)
1	Trần Lê Hoàng		API upload tệp/ảnh, tìm kiếm tin nhắn/user, realtime TCP socket server, FE file/image render, Search Bar, Global search, DB attachments	100 %
TỔNG CỘNG	100%

* Nhiệm vụ chính: Phát triển lõi truyền nhận thời gian thực, đính kèm tệp tin và tìm kiếm.
* Backend (FastAPI & Python Socket):
  - Xây dựng API upload ảnh/tệp tin: POST `/upload/image`, POST `/upload/file` (lưu trữ file tải lên vào thư mục `uploads/` trên server)
  - Viết API tìm kiếm người dùng toàn cầu theo tên hoặc SĐT: GET `/search/users`
  - Viết API tìm kiếm tin nhắn trong lịch sử chat: GET `/search/messages`
  - Hỗ trợ xây dựng Server TCP Realtime chạy ngầm ở port 9000 (`realtime/server.py`) để làm nền tảng socket giao tiếp sau này.
* Frontend (React):
  - Tích hợp nút upload ảnh/file ở khung nhập tin nhắn (`MessageInput`).
  - Xử lý render hiển thị tệp tin dạng file đính kèm hoặc ảnh trực quan trong khung chat.
  - Phát triển tính năng tìm kiếm tin nhắn trong cuộc trò chuyện (Search Bar ở header).
  - Xử lý tìm kiếm người dùng toàn cầu khi nhập vào ô tìm kiếm trên cùng của Sidebar.
* Database:
  - Bảng `attachments`: (nếu cần quản lý tệp đính kèm nâng cao: `id`, `message_id`, `file_path`, `file_type`).
